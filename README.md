# Escape from SpaceWar · 3D 太空战斗与星域战役原型

基于 TypeScript、Vite 和 Three.js 的确定性太空战斗模拟器，并包含可玩的星域 Roguelike 战役。

单场战斗使用固定 tick 和 seed 驱动；战役层在此基础上提供结构化星域探索、持久舰队、资源与货舱、打捞维修、撤退撤离、指挥官生涯、组织身份和模块化科技。

## 当前功能

### core-v4 单场战斗

- 3 舰种 × 4 改型，共 12 种合法舰船组合
- 固定 30 tick/s 的确定性模拟
- Three.js 3D 观看、自动镜头、暂停与倍速
- Replay Code、舰队库、战前分析、平衡实验室和战舰图鉴

### V0.6～V0.7.1 星域与持久舰队

- 确定性分层星域、战争迷雾、扫描、资源、信号、hazard 与威胁
- 持久舰船 ID、组件 HP、失能、拖曳、拆解和放弃
- 有容量与重量限制的货舱
- 战后打捞、敌舰回收、救援舰和战地维修
- 战前部署、风险评估、规避、手动撤退和自动撤退
- 普通与紧急跃迁、主动抛货、星域结算和第三星域胜利
- 七层航道、区域主题、多路线和移动端可读性优化

### V0.8～V0.8.1 指挥官生涯

- 新战役设置指挥官姓名和初始专长
- 指挥、战术、后勤、意志四项属性
- 两个不重复且由 seed 确定的特质
- 战斗、探索、后勤、生存四类领域经验
- 疲劳、动摇、负伤、伤病、治疗和自然恢复
- 确定性招募、最多三名候补指挥官和继任规则
- 招募频率、治疗成本、创伤阈值和窄屏布局已完成试玩收口

### V0.9 组织身份、政体与模块化科技

#### 组织创建

- 四种组织原型：远征舰队、军事共同体、商贸联合体、流亡殖民团
- 五种政体：军事委员会、舰长议会、商业董事会、技术官僚制、紧急统制局
- 从秩序、自由、生存、扩张、知识、利润、团结中选择两项不同价值观
- 组织稳定度和民间、军事、边疆三类声望
- 相同 seed 与创建选项得到相同组织 ID 和初始状态

#### 研究与科技

研究资源分为：

- 航行数据
- 工程数据
- 战术数据
- 社会数据

研究数据来自扫描、采集、信号、战斗、打捞、维修、治疗和跃迁。

六项首版科技模块：

| 科技 | 主要作用 |
|---|---|
| 高效跃迁校准 | 降低跃迁燃料消耗 |
| 模块化货舱 | 增加 4 点货舱容量 |
| 战地维修协议 | 降低维修威胁并增加工程数据 |
| 深空传感阵列 | 降低扫描威胁并提高规避率 |
| 撤退协同协议 | 提高规避并降低战败稳定度损失 |
| 创伤救治程序 | 降低治疗成本 |

- 每个组织初始解锁并装配一个原型科技
- 默认拥有两个科技槽
- 科技需要研究资源解锁，再装配到槽位后生效
- 货舱超出基础容量时不能卸下模块化货舱

#### 组织事件

- 跨星域后生成确定性的组织决策
- 事件可围绕救援配额、航线安全和遗物处置展开
- 部分选项要求特定价值观
- 事件可以改变稳定度、声望、资源、威胁和研究数据
- 待处理组织事件会阻塞常规星域行动
- 组织稳定度归零会导致战役失败

V0.9 只修改战役层，不改变冻结的 core-v4 舰船模板、默认战斗数值、AI 决策和黄金回放。

## 确定性原则

1. 战斗使用固定 30 tick/s。
2. 所有影响结果的随机行为使用 seed 派生 PRNG。
3. 模拟和战役逻辑禁止使用非确定性时间或随机源决定结果。
4. 相同配置、seed 和行动序列必须得到一致结果。
5. 星域、战利品、hazard、敌军、规避、招募、组织事件和跃迁损伤均由稳定输入派生。
6. 指挥官经验由持久化历史幂等重建。
7. 组织创建、事件、研究和科技操作进入 Campaign Code 深层校验。
8. 渲染层只消费状态，不能反向修改模拟结果。

## Code 格式

### Replay Code

- 当前版本：`v0.5`
- 规则集：`spacewar-core-v4`
- 用于普通单场战斗回放

### Fleet Code

- `type: "spacewar-fleet"`
- 描述舰队、阵型和 doctrine

### Campaign Code

- `type: "spacewar-campaign"`
- 当前版本：`0.3`
- 保存星域、资源、货舱、舰队、舰损、指挥官、候补、组织、政体、价值观、研究、科技和全部待处理状态
- 支持导入 `0.1` 和 `0.2` Campaign Code，并迁移为 `0.3`

### Campaign Log

- `type: "spacewar-campaign-log"`
- 当前格式：`1.1`
- 包含指挥官、候补名单、组织档案、研究科技、资源、舰队、当前位置和完整历史

## 项目目录结构

```text
src/
├── sim/                  # core-v4 战斗模拟、回放和测试
├── campaign/
│   ├── cargo/            # 货舱和物品
│   ├── commander/        # 指挥官创建、生涯、健康、招募和继任
│   ├── deployment/       # 战前部署
│   ├── extraction/       # 撤离规划和跃迁结算
│   ├── fleet/            # 持久舰队、遭遇和 Battle Adapter
│   ├── organization/     # 组织、政体、事件、研究和科技
│   ├── repair/           # 战地维修和重新启用
│   ├── salvage/          # 战后打捞和舰体回收
│   └── sector/           # 星域、情报、事件和威胁
├── render/               # Three.js 渲染
└── ui/                   # 单场战斗与战役 UI
```

## 已知限制与暂不实现内容

- 只支持一支玩家舰队；组织不能拥有独立舰队或基地
- 声望目前只影响组织档案和事件结果，尚无完整外部派系外交
- 科技采用少量模块槽，不是大型传统科技树
- 没有市场价格模拟、贸易路线或组织内部派系政治
- 没有随机装备词条、舰船制造或完整舰员个体系统
- 战役战斗暂不支持 seek 或 Replay 分享
- 无后端、账号或联网功能
