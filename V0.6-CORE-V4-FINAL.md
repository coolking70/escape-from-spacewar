## 3. CombatState 最终状态机

7 个状态，优先级（高→低）：**destroyed > escaped > disabled > retreating > critical > damaged > normal**。

- **HP 退化**：normal → damaged（总 HP≤0.6 或核心≤0.5）→ critical（总 HP≤0.25 或核心≤0.3）。
- **撤退**：任意活跃态设置 `retreatStartedTick` 且无失能 → retreating。
- **失能**：`mobilityDisabled || weaponsDisabled || (sensorsDisabled && !hasCloseEnemy)` → disabled（优先于 retreating/critical）。
- **脱战（终态）**：retreating 抵达边界设置 `escapedTick` → escaped（不可逆；保存舰队价值）。
- **摧毁（终态）**：核心摧毁由 sim 显式置 `combatState='destroyed'`（不可逆；损失）。
- 终态优先：一旦 destroyed/escaped，`computeCombatState` 立即短路返回，拒绝任何回退（escaped→destroyed、disabled→escaped、destroyed→normal 等均非法）。

判定函数（`shipFlags.ts`，纯 `combatState`）：
`isPresentOnBattlefield = isStructurallyAlive = (≠destroyed && ≠escaped)`；`isTargetable = (≠destroyed && ≠escaped)`（disabled 仍可被攻击）；`isCombatCapable = normal/damaged/critical/retreating`。

## 4. Fleet Value 计算方式

价值口径**纯由 `CombatState` 派生**，**互斥**累加，绝不再读 `alive`。单舰原始成本 `cost = getVariantDef(variant).cost`。

| CombatState | operational（作战价值） | decision（判定价值） | 归类价值 |
|-------------|:--:|:--:|---------|
| normal / damaged / critical / retreating | 100% cost | 100% cost | remainingOperationalValue |
| escaped | 0 | 100% cost | escapedValue |
| disabled | 0 | 50% cost | disabledValue |
| destroyed | 0 | 0 | destroyedValue |

**守恒不变量（每队，原始成本口径）**：
```
initialFleetCost = Σ cost
destroyedValue + disabledValue + escapedValue + remainingOperationalValue = initialFleetCost
remainingDecisionValue = disabledValue×0.5 + escapedValue×1 + remainingOperationalValue×1
```
- `remainingOperationalValue`：仍在场且具战斗力的作战价值（仅活跃态）。
- `remainingDecisionValue`：超时/点数裁决用的“保存舰队价值”（escaped=100%、disabled=50%、destroyed=0%）。
- 互斥：escaped **不再**计 destroyed；disabled 不计 operational；七状态对每舰恰取其一。

> 旧 `getShipPointValue` 保留为 decision value 的冻结别名（黄金指纹用），语义与 `getShipDecisionValue` 完全一致。

## 5. PointsDecision 计算方式

`decideVictory(state)`（`combatState.ts`）按双方 `combatCapable` 计数与 decision value 判定：

1. 统计每队 `capable`（`isCombatCapable` 为真）、`destroyed`、`escaped`、`value = Σ getShipPointValue`（= decision value）。
2. **双方均无可战斗单位**（`capableA===0 && capableB===0`）：`value` 高者胜 → `pointsDecision`；相等 → `draw`。
3. **仅一方为 0**：另一方胜；胜方原因——对方全灭→`annihilation`；对方有 escaped→`retreat`；否则→`combatDisabled`。
4. **双方仍有战斗力**（仅 maxTicks 到达）：`value` 高者胜 → `timeout`；相等 → `draw`。

价值修正（本阶段关键修复）：escaped 由旧“alive=false→0”修正为 **100% cost** 计入 decision value，使“保存舰队价值”真正参与超时/点数裁决（v4-small-balanced 因此由 draw 变为 B 胜/timeout）。

## 6. VictoryReason 判定流程

```
每 tick 末 / maxTicks 到达 → decideVictory(state)
  ├─ capableA==0 && capableB==0  →  value 高者胜(pointsDecision) / 平(draw)
  ├─ capableA==0                 →  B 胜；A 全灭→annihilation / A 有 escaped→retreat / 否则 combatDisabled
  ├─ capableB==0                 →  A 胜；B 全灭→annihilation / B 有 escaped→retreat / 否则 combatDisabled
  └─ 双方仍有战斗力(超时)        →  value 高者胜(timeout) / 平(draw)
判定完全基于 combatState / isCombatCapable / decision value，不读 alive。
```

VictoryReason 枚举：`annihilation`（全歼）/ `combatDisabled`（战术失能）/ `retreat`（敌方撤退）/ `timeout`（超时点数）/ `pointsDecision`（点数判定）/ `draw`（平局）。
